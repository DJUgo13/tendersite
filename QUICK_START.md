#  Быстрый запуск проекта

## 1. Активация виртуального окружения

```bash
# Windows PowerShell
.\venv\Scripts\Activate.ps1

# Или Command Prompt
venv\Scripts\activate.bat
```

## 2. Установка зависимостей (если нужно)

```bash
pip install -r requirements.txt
```

## 3. Применение миграций

```bash
python manage.py migrate
```

## 4. Создание суперпользователя (администратора)

```bash
python manage.py createsuperuser
```

При создании используйте:
- **Username**: `admin`
- **Email**: (можно оставить пустым)
- **Password**: `B5200AB@as12`

## 4.1. Применение миграций (ВАЖНО!)

После создания суперпользователя нужно применить миграции для новых моделей:

```bash
python manage.py makemigrations
python manage.py migrate
```

## 4.2. Создание профиля для администратора

После миграций создайте профиль для администратора:

```bash
python manage.py shell
```

В shell выполните:
```python
from django.contrib.auth.models import User
from core.models import UserProfile

user = User.objects.get(username='admin')
UserProfile.objects.get_or_create(user=user, defaults={'role': 'admin'})
print("Профиль администратора создан!")
```

## 5. Запуск сервера разработки

```bash
python manage.py runserver
```

Сервер запустится на `http://127.0.0.1:8000/`

## 6. Доступ к системе

- **Админ-панель**: http://127.0.0.1:8000/admin/
- **Вход для менеджера**: http://127.0.0.1:8000/core/
- **Панель руководства**: http://127.0.0.1:8000/core/leadership/ (требуется роль leadership)
- **Мобильная версия**: откройте на мобильном устройстве или в DevTools (F12 → Ctrl+Shift+M)

### Роли пользователей:
- **Администратор** (`admin`) - полный доступ, включая админ-панель
- **Менеджер** (`manager`) - участие в тендерах, подача ставок
- **Руководство** (`leadership`) - только просмотр отчётов и статистики

## 7. Первоначальная настройка

1. Войдите в админ-панель (`/admin/`)
2. Создайте пользователя для компании:
   - Перейдите в "Users" → "Add user"
   - Создайте пользователя (например, `manager1`)
3. Создайте компанию:
   - Перейдите в "Companies" → "Add company"
   - Укажите ИНН, название, выберите созданного пользователя
4. Создайте тендер:
   - Перейдите в "Tenders" → "Add tender"
   - Заполните данные и добавьте направления

##  Тестирование мобильной версии

1. Откройте Chrome DevTools (F12)
2. Нажмите Ctrl+Shift+M (режим устройства)
3. Выберите устройство (iPhone, Android и т.д.)
4. Обновите страницу

##  Важно

- Для работы email уведомлений настройте SMTP в `settings.py` или используйте режим разработки (письма в консоль)
- Иконки PWA нужно создать (см. `README_ICONS.md`)
- Для генерации PDF протокола нужен пакет `reportlab` (установится через `requirements.txt`)

##  Автоматическая проверка финальных таймеров (направления)

Команда для ручного запуска проверки таймеров:

```bash
python manage.py check_timers
```

Для “до победы” в продакшене настройте запуск этой команды по расписанию (например, каждые 1–2 минуты через Task Scheduler / cron).
